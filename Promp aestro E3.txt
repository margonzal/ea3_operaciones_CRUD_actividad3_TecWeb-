REQUISITOS OBLIGATORIOS

BASE DE DATOS

Motor:
MySQL

Nombre de la base:
monitoreo_mecanico

Tabla:
sensores

Campos obligatorios:

id INT AUTO_INCREMENT PRIMARY KEY
nombre VARCHAR(100)
tipo ENUM('temperatura','presion','vibracion')
valor DECIMAL(10,2)
unidad VARCHAR(20)
ubicacion VARCHAR(120)
fecha_registro DATETIME DEFAULT CURRENT_TIMESTAMP

La estructura debe seguir exactamente la guía de la actividad.

DATOS DE PRUEBA

El archivo schema.sql debe incluir al menos 4 registros de prueba.

Distribución recomendada:

- 2 sensores de temperatura
- 1 sensor de presión
- 1 sensor de vibración

Estos registros deben permitir que el dashboard muestre información desde el primer momento.

API REST

Implementar exactamente los siguientes endpoints:

POST /api/sensores

GET /api/sensores

GET /api/sensores?tipo=temperatura

PUT /api/sensores/:id

DELETE /api/sensores/:id
